# 创意视频综合套件

## 定位

本 Skill 是视频创作的统一主入口，根据用户需求自动路由到两大视频创作分支：

```text
用户需求 → 需求分类判断 → ┬─ 短剧/剧情视频分支 → 剧本→分镜→资产→关键帧→视频
                           └─ 商业视频分支 → UGC/Marketing/Corporate → 视频
```

**两大分支：**

| 分支 | 适用场景 | 核心参考文件 |
|---|---|---|
| **短剧/剧情视频** | 短剧、动画、微电影、剧情视频、影视化、动态漫、预告片、有完整故事线的视频 | `references/drama/` 目录 |
| **商业视频** | 产品广告、UGC口播、带货视频、企业宣传、品牌形象片、产品功能介绍、营销视频 | `references/commercial/` 目录 |

## 核心通用规则

### 视频生成门禁

**当用户要求生成视频时，不要立即开始生成，也不要立即调用视频生成工具。**

必须先澄清并确认以下基础参数：

- **时长**：视频长度，例如 `5s`、`10s`、`15秒`。
- **比例**：
  - 文生视频支持：`16:9`、`4:3`、`1:1`、`3:4`、`9:16`、`21:9`
  - 图生视频（r2v模式）支持：`16:9`、`4:3`、`1:1`、`3:4`、`9:16`、`21:9`
  - 图生视频（f2v模式）仅支持保持原比例，如果用户要求调整，必须跟用户说明
- **生成详细内容**：画面主体、动作、场景、风格、镜头运动、情绪氛围、文字、声音或其他限制。
- **生成数量**：一次最多生成2个视频，超出必须对用户进行澄清：一次仅能生成两个视频。

### 确认机制

**只有在用户明确确认参数后，才可以继续生成。** 无论用户是否已经一次性提供完整参数，都必须先输出一次参数摘要并等待确认；不能因为信息看似完整就直接进入生成。

明确确认可以是：`确认`、`可以生成`、`开始生成`、`没问题`、`yes`、`looks good`、`ok` 等表达。

**确认必须是本轮任务确认**：每一个新视频任务、换产品、换 brief、换平台、换时长、换风格，都必须重新输出确认摘要并等待用户确认；历史任务的确认不得复用。

`好`、`OK`、`嗯` 这类短回复只有紧跟确认摘要时才算确认；如果它们出现在新需求之后，只能视为新需求内容或继续沟通，不能直接生成。

### 即时输出原则

在资产设定（如角色设定图、场景图等）和视频生成阶段，**每生成完一张图或一个视频，必须立即将其展示并返回给用户**。绝对禁止等待所有图片或所有视频全部生成完毕后再统一打包返回。

### 图片质量审查原则

在完成任何一个阶段的图片生成之后，都应检查图片是否存在问题：

- **人设图审查**：重点审查左侧面部特写是否是正脸，三视图是否与左侧面部特征保持一致，三视角是否是同一个人物。
- **关键帧审查**：重点审查画面是否和对应剧情对的上，人物、场景、道具是否和资产设定图中的对应特征对的上。

### 交付规范

所有视频生成结果，无论文生视频/图生视频、UGC/marketing/corporate、短剧，交付前都必须调用 `notify_hunman` 整理输出并展示视频；不得以"非飞书内容""在线 URL 可直接回复""链接可展示""Markdown/HTML video 标签可播放""用户偏好直接交付"等理由跳过。

`notify_hunman` 入参必须包含可渲染的视频资源本体：`video_url/output_url/url/result_url` 或本地视频路径；不得只传总结文案、HTML/Markdown `<video>` 片段或裸链接文本。

只有确认 `notify_hunman` 已正常上屏视频后，才可以输出 `视频已生成` 或成功格式；如未正常上屏，必须用同一视频资源重试一次，仍失败再说明展示失败并保留链接/路径排查。

## 可选后端：Agnes AI

本 Skill 支持使用 **Agnes AI** 作为可选的生成后端。Agnes AI 是新加坡 Sapiens AI 团队开发的多模态 AI 平台，提供文本、图像、视频三大模态的 API 服务，目前无限期免费开放。

### 什么时候使用 Agnes AI

- **需要免费额度时**：Agnes AI 目前完全免费，适合批量生成或测试
- **需要高分辨率图像时**：支持最高 4K 图像生成
- **需要多图合成时**：支持多张参考图组合生成新图像
- **需要 OpenAI 兼容接口时**：方便迁移现有代码或工具链

### 模型对应关系

| 创作阶段 | 系统内置工具 | Agnes AI 模型 |
|---|---|---|
| 剧本/分镜/提示词 | 内置文本能力 | `agnes-2.0-flash` |
| 角色/场景资产图（文生图） | `image_gen` | `agnes-image-2.1-flash` |
| 关键帧/图像编辑（图生图） | `image_edit` | `agnes-image-2.0-flash` |
| 视频生成 | `text_to_video` / `image_to_video` | `agnes-video-v2.0` |

### 配置方法

**获取 API Key**：
1. 访问 [Agnes AI 控制台](https://platform.agnes-ai.com) 注册账号
2. 进入 API Key 管理页面
3. 创建并复制你的 API Key

**使用方式**：
- **环境变量**：设置 `AGNES_API_KEY` 环境变量
- **脚本调用**：使用 `scripts/` 目录下的 Python 脚本
- **直接调用**：使用 `scripts/agnes_ai_client.py` 中的 `AgnesAIClient` 类

### 脚本工具

本 Skill 提供了以下脚本工具（位于 `scripts/` 目录）：

| 脚本 | 用途 |
|---|---|
| `agnes_ai_client.py` | Agnes AI API 客户端类，提供完整的 Python 接口 |
| `generate_with_agnes.py` | 便捷命令行工具，支持图像、视频、角色设定图生成 |

**命令行工具示例**：

```bash
# 文生图
python scripts/generate_with_agnes.py image -p "一只可爱的猫" -s 1024x1024

# 图生图
python scripts/generate_with_agnes.py image -p "改成赛博朋克风格" -r https://example.com/img.png

# 文生视频（10秒）
python scripts/generate_with_agnes.py video -p "海边日落" --duration 10

# 图生视频
python scripts/generate_with_agnes.py video -p "人物慢慢转头" -i https://example.com/frame.png

# 生成角色设定图（快捷方式）
python scripts/generate_with_agnes.py character -n "小明" -d "年轻男性，黑色短发" --style "真人电影"
```

完整的 API 文档和使用说明请参考：`references/agnes-ai-api.md`

包含内容：
- 文本模型 API 详解
- 图像模型 API 详解（文生图、图生图、多图合成）

### 使用建议

**混合使用策略**（推荐）：
1. **策划阶段**：用 Agnes AI 文本模型生成剧本、分镜、提示词
2. **资产阶段**：用 Agnes AI 图像模型生成角色、场景、道具图
3. **关键帧阶段**：用 Agnes AI 图生图能力生成关键帧
4. **视频阶段**：根据需求选择 Agnes AI 或系统内置视频工具

**注意事项**：
- Agnes AI 视频生成是异步模式，需要轮询等待（通常 2-5 分钟）
- 图像 URL 需要是公网可访问的 HTTPS 链接
- API Key 请妥善保管，不要暴露在公开代码中

## 需求分类判断

根据用户输入，判断属于哪类视频创作：

### 短剧/剧情视频（走 Drama 分支）

**触发关键词**：短剧、剧本、分镜、人设、角色设定、剧情、故事、微电影、动画、动漫、动态漫、影视化、电影、预告片、连续剧、分集、剧集、人物设定、场景设定、资产设定、关键帧

**特征**：
- 有完整故事线和角色发展
- 需要剧本创作或分镜设计
- 涉及多个人物和场景
- 需要保持角色形象一致性
- 通常时长较长或有多集

### 商业视频（走 Commercial 分支）

**触发关键词**：广告、产品视频、带货、UGC、口播、测评、开箱、教程、企业宣传、品牌形象、产品介绍、营销视频、TVC、信息流、种草、达人视频、商务视频、招商片、路演片

**特征**：
- 以产品/品牌/企业为核心
- 通常有明确的营销目标
- 可能包含口播、旁白、解说
- 单条视频，时长较短（通常15秒左右）
- 更注重产品展示和转化效果

### 模糊需求处理

如果用户需求不明确，先询问 1-2 个关键问题来确定方向：
- "请问您是想做有剧情故事的短剧/动画，还是产品广告/宣传类视频？"
- "这个视频主要是用于故事叙事，还是产品推广/企业宣传？"

## 短剧/剧情视频分支

当判断为短剧/剧情视频时，按以下流程推进。

### 生产流程

```text
planner → scriptwriter → storyboard → assets → frame → prompt → 视频生成
```

`planner` 就是当前主 `SKILL.md`。其它阶段按需读取 refs：

| 阶段 | 读取文件 |
|---|---|
| 剧本创作 | `references/drama/scriptwriter.md` |
| 分镜切分 | `references/drama/storyboard.md` |
| 资产设定 | `references/drama/assets.md` |
| 关键帧生成 | `references/drama/frame.md` |
| 视频提示词 | `references/drama/prompt.md` |

**不得跳阶段**。只要发现上游依赖缺失，就停在最早缺失阶段继续处理。

### 阶段判断

按顺序检查，并停在第一个缺失依赖：

1. 只有 idea、标题、简短梗概、不完整故事或松散角色概念时，开始剧本创作。
2. 有完整故事但没有按集/片段/镜头切分时，开始分镜切分。
3. 有分镜但没有可用角色、场景、道具资产引用或详细视觉基准时，开始资产设定。
4. 有资产和分镜但没有片段关键帧图时，开始关键帧生成。
5. 有关键帧但没有最终视频提示词执行包时，开始视频提示词生成并生成对应视频。

### 分阶段澄清

| 阶段 | 需要澄清 |
|---|---|
| planner | 目标、产物形式、受众、全流程范围、已有素材、希望从哪个阶段开始。 |
| scriptwriter | 故事主题、题材、目标时长或集数、主角、核心冲突、结局方向、情绪基调、禁用内容。 |
| storyboard | 要处理的集数、单集时长、片段结构、镜头密度、台词/字幕/声音要求。 |
| assets | 角色、场景、道具、视觉风格、参考图用途、一致性锚点、音色或台词锚点。 |
| frame | 哪一集/哪些片段、首尾关键帧要求、场景引用、角色引用、构图、光影、角色物理状态连续性。 |
| prompt | 工具模式、时长、比例、首帧、参考资产、时间戳动作规划、台词/声音、负向约束、批次数量。 |

### 路由规则

- 当前阶段是剧本创作时，读取 `references/drama/scriptwriter.md`。
- 当前阶段是分镜切分时，读取 `references/drama/storyboard.md`。
- 当前阶段是资产设定时，读取 `references/drama/assets.md`。
- 当前阶段是关键帧生成时，读取 `references/drama/frame.md`。
- 当前阶段是最终视频生成时，读取 `references/drama/prompt.md`。

**每阶段完成后都必须用户确认：**
在剧本、分镜、资产、关键帧的任何一个阶段完成输出后，**必须停下来，并明确询问用户是否满意、是否需要修改**。绝对禁止在没有获得用户明确的"确认/没问题/继续/yes/ok"等许可下，擅自读取下一个 Reference 文件并自动推进到下一阶段。

### 角色形象变化资产更新原则

随着剧情进展，如果人物形象发生较大变化（如换装、发型变化、受伤/毁容、年龄阶段变化、身份伪装、盔甲/制服/礼服等关键造型变化），必须先重新生成对应阶段的角色资产设定图，并经用户确认后，才能继续生成后续关键帧或视频。禁止沿用旧资产图硬套新造型。

## 商业视频分支

当判断为商业视频时，按以下流程推进。

### 工作流程

1. 识别用户是否有视频生成意图。
2. 判断请求属于普通视频、产品广告视频、口播/旁白视频、UGC、marketing、corporate/企业宣传商务视频，或这些类型的组合。
3. 提取用户已经给出的时长、比例、生成详细内容、产品信息、口播/旁白信息和 ref 信息。
4. 如果命中产品广告、UGC、marketing 或 corporate/企业宣传商务视频，先执行「产品/企业资料门控」。
5. 如果基础参数、产品信息或口播信息缺失或模糊，先向用户追问缺失项；如果用户已给全参数，也必须继续进入确认摘要，不能直接生成。
6. 如果包含口播、旁白或解说，还必须明确最终台词来源：用户提供、ref 提取、或由 AI 起草；最终台词必须出现在确认摘要中。
7. 信息补齐或用户选择"无产品参考直接生成"后，向用户汇总最终视频生成参数并请求确认。
8. 如果用户修改任何参数，更新摘要并重新请求确认。
9. 用户明确确认后，才调用视频生成工具或进入视频生成流程。
10. 进入 UGC/marketing/corporate ref 策略前，先执行「分镜图片/关键帧门控」；没有明确图片需求时，只输出分镜表，不生成分镜图或关键帧图。

### 产品/企业资料门控

命中产品广告、UGC、marketing 或 corporate/企业宣传商务视频时，默认必须先有产品上传、企业资料、品牌资产或可用 ref，再进入对应 ref 策略。

如果用户没有提供任何产品、企业资料、品牌资产或可用 ref：

1. 先澄清一次：请用户上传产品/企业/品牌参考，或明确选择"无参考直接生成/直接生成/按概念 brief 生成"。
2. 用户选择上传资料时，等待资料后再汇总确认。
3. 用户明确选择直接生成时，允许继续，但确认摘要必须写明：`参考资料：无，按概念/文字 brief 生成，产品外观、企业视觉和品牌资产不保证真实一致`。

### 分镜图片/关键帧门控

默认交付是**分镜表**，不是分镜图片。主 MD 和所有 ref 策略都必须遵守：

- 用户只说"分镜""分镜脚本""分镜表""storyboard""shooting script"时，只输出表格或文字分镜，不生成图片，不调用图像生成/关键帧生成工具。
- 只有用户明确说"分镜图""分镜图片""关键帧图""关键帧图片""首帧图""每个镜头出图""画面图""storyboard images""keyframe images"等图片交付词时，才可以生成分镜图片、关键帧图或首帧图。
- 用户要最终视频但没要分镜图片时，流程是：分镜表 → 视频 prompt → 视频工具；不得插入分镜图生成步骤。
- 分镜表中的台词/旁白/声音为必填字段：可以写完整原文，也可以写"无台词/无旁白，仅 BGM + 环境声"；不得缺失、留空或默认省略。

### ref 路由策略

策略 ref 路由：

- **UGC 口播路由**：用户要口播、真人/数字人讲解、种草、测评、开箱、教程、批量达人视频、UGC 参考复刻时，读取 `references/commercial/ugc-talking-video-ref.md`。用户提到信息流、信息流带货、信息流广告、带货视频、达人带货或口播带货时，优先读取此文件。
- **Marketing 无分镜路由**：用户要产品营销广告、商品广告、单品种草、产品展示大片、无分镜图广告片，且目标以单品消费转化、产品视觉广告、平台投放为主，并且未提信息流、带货、口播、达人或真人讲解时，读取 `references/commercial/product-marketing-ad-video-no-storyboard-ref.md`。
- **Corporate 商务视频路由**：用户要产品功能介绍、企业宣传片、企业产品功能宣传、商务视频、品牌形象片、企业介绍视频、招商片、路演片、会议开场片、客户案例片、雇主品牌视频、服务介绍片、B2B 广告片或公司宣传短片时，读取 `references/commercial/corporate-business-video-ref.md`。

**冲突处理**：
- 用户提到信息流、信息流带货、信息流广告、带货视频、达人带货或口播带货时，不走 Marketing ref；除非用户明确说不要真人/达人/口播、只要纯产品广告大片，再按确认后的目标改走 Marketing ref。
- 同时包含产品广告和真人口播时，优先看用户目标：以真人推荐/口播可信感/信息流带货为主走 UGC ref；以产品大片/广告级镜头为主且未提信息流/带货/口播时走 Marketing ref。
- Marketing 与 Corporate 冲突处理：以产品大片/广告级镜头为主且未提信息流/带货/口播时走 Marketing ref；以企业能力、公司介绍、商务信任、招商路演、客户案例、雇主品牌或会议开场为主时走 Corporate ref。

### 口播和旁白要求

当用户要求视频包含口播、旁白、解说、主播念词、真人讲解、voiceover、host speech、talking-head delivery 或类似 spoken content 时，必须在生成前明确完整台词。

- UGC 口播进入视频 prompt 时必须用 `{具体台词}` 包裹，只作为 `Monologue`、`audio_voiceover` 或 `Audio` 的声音/口型内容，不得作为字幕、caption、lower-third 或画面文字。
- Marketing 视频默认添加完整旁白台词。诗性旁白和 slogan 是固定搭配，不是可选关系；只有用户明确要求不要旁白或纯音乐形式时才取消旁白。
- Corporate/企业宣传商务视频必须包含完整旁白台词；只有用户明确要求无旁白或纯音乐时才遵从该意图。

## 全局检索触发规则

- 用户提到"豆包"，并要求合影、互动、生成豆包形象、豆包角色/IP/头像/表情时，必须调用搜索工具检索豆包 APP 角色形象；不能仅凭记忆生成豆包视觉。
- 出现时效词，如"爆火、最近、最新、新闻、今年、现在、今天"，且涉及品牌、公司、产品、IP、人物、地点、赛事、节日、热点时，必须调用搜索工具获取最新信息；不能把过期记忆当作当前事实。
- 检索结果只能用于补充事实、视觉参考、平台语境和风险判断；最终生成参数仍必须回到本 MD 的澄清确认流程，未经用户确认不得直接生成。

## 工具门禁

以宿主环境实际可用工具为准。旧版工具名按下表理解：

| 旧工具名 | 当前含义 |
|---|---|
| `text_to_image` | 文生图 |
| `image_to_image` | 基于参考图的图像编辑 / 图像生成 |
| `image_to_video` | 基于图片参考的视频生成 |

**工具调用失败处理：**
当阶段规则要求使用图像编辑/I2I 或视频生成/I2V 时，如果实际调用失败，必须优先重新尝试调用对应工具，仍失败再停下来说明失败原因与所需补充信息。禁止为了绕过失败而擅自降级改用纯文生图或纯文生视频，这会丢失参考图约束并破坏角色、场景、道具一致性。

## 完成检查

声明完成前必须确认：

- 没有跳过上游阶段。
- 视频生成前已完成必要澄清并获得明确确认。
- 比例、时长、模式、数量、参考素材和限制已记录。
- 角色、场景、道具和关键帧连续性已保持。
- 剧情导致人物形象明显变化时，已补充并确认新的角色资产设定图。
- 应使用 I2I / I2V 的任务没有因工具调用失败而降级为 T2I / T2V；如失败，已重试对应工具或明确说明失败原因。
- 单批生成视频数量不超过 2 个。
- 资产和视频的输出必须遵循"即时输出"原则，生成一个返回一个，不得囤积。
- 所有视频生成结果交付前都已调用 `notify_hunman` 整理输出并展示视频。
