# 关键帧生成 Ref

当剧本、分镜表和资产已获得用户确认/由用户提供，需要生成片段关键帧图时读取本文件。

## 目标

把确认后的资产和每集 6 个片段的分镜，转化为关键帧图 prompt 和关键帧图片。

每个 90 秒短剧单集默认生成 7 张关键帧：

1. `SEG01_START`：片段一第 1 秒画面。
2. `SEG01_END`：片段一最后 1 秒画面。
3. `SEG02_END`：片段二最后 1 秒画面。
4. `SEG03_END`：片段三最后 1 秒画面。
5. `SEG04_END`：片段四最后 1 秒画面。
6. `SEG05_END`：片段五最后 1 秒画面。
7. `SEG06_END`：片段六最后 1 秒画面。

## 澄清策略

关键帧生成前确认：

- 项目名、集数、画幅比例、统一画风。
- 要处理的具体集数 / 片段。
- 是否已有 6 个 15 秒片段。
- 角色、场景、道具参考图是否可用。
- 是否使用短剧默认竖屏比例 `9:16`。

如果缺少必要角色脸部 / 服装、场景空间 / 光影或关键道具资产，停止当前关键帧任务，返回 `assets.md`。

## 关键帧工具调用硬约束

- 关键帧生成默认必须使用 `image_to_image` / I2I。
- 只要已有角色、场景、道具资产图，生成关键帧时必须将相关资产图作为工具参考图参数传入，不能只在 Prompt 文本中描述或编号。
- 禁止在已有资产图的情况下使用纯 `text_to_image` 生成关键帧。
- 每次生成前必须先列出本张关键帧实际传入的参考图清单：动态编号（`[img0]`、`[img1]`、`[img2]`...）、资产名称、URL / 本地路径、用途。
- 生图 I2I Prompt 只能使用 `[img0]`、`[img1]`、`[img2]`... 引用输入图，禁止使用 `@图片`、`图1`、`参考图1` 等格式。
- 如果找不到必要资产图，必须停止关键帧生成并返回资产阶段补齐，不能自行脑补。

## 关键帧命名

```text
EP01_SEG01_START.png
EP01_SEG01_END.png
EP01_SEG02_END.png
EP01_SEG03_END.png
EP01_SEG04_END.png
EP01_SEG05_END.png
EP01_SEG06_END.png
```

## 提取要求

每张关键帧都要：

- 找到对应片段和具体时刻。
- **参考图动态编号**：从已确认的角色、场景、道具资产中提取最合适的参考图，并作为 `image_to_image` / I2I 的工具参考图参数传入。将实际选出的参考图按顺序编号为 `[img0]`、`[img1]`、`[img2]`...，并在生图 Prompt 中严格使用这些编号指代。**绝不传入无关资产图**。
- 保持当前物理状态：湿衣服、伤痕、污渍、手持道具、面部紧张、光影、天气、色温等。
- 写清动作结果、表情、姿势、构图、景别、机位角度和剧情连续性。
- **全局风格锚点**：Prompt 开头必须加上整个项目统一的风格限定词（如 真人电影风格）。

## I2I 生图指令撰写规范

每张关键帧的 I2I Prompt 必须按“编辑目标 + 保留项 + 编辑项 + 空间线性描述 + 风格质量约束 + 画幅比例”组织，最终合并为一段连贯的话，不要输出“编辑目标：”“保留项：”等小标题。

- **编辑目标**：直接说明当前关键帧要基于哪些 `[imgX]` 完成什么画面生成，明确来源图片、目标状态和作用范围。
- **保留项**：明确参考图中必须保持不变的主体身份、角色脸部特征、服装结构、场景空间、构图关系、透视角度、光影方向、材质细节和道具特征。禁止只写“保持不变”“其他不变”“参考原图”。
- **编辑项**：逐项写清需要新增、替换、重绘、重排或状态变化的对象，例如角色动作、表情、站位、湿衣服、伤痕、手持道具、场景交互区变化等。
- **空间线性描述**：按从背景到主体、从主体到局部、从主要编辑区域到次要编辑区域的顺序描述画面；每个区域写清位置、原参考内容、保留项、编辑动作、目标结果和视觉约束。
- **多图约束**：若传入多张参考图，必须分别说明 `[img0]`、`[img1]`、`[img2]` 的用途和保留内容，不得使用“同上”“其余图片一样处理”等概括。
- **文字约束**：如关键帧涉及可见文字，必须逐条写清原文案与目标文案，所有文字用中文双引号 `""` 框定。
- **风格与质量约束**：写清整体风格、光线、色调、材质、阴影、真实度和融合要求，确保修改区域与参考图自然融合。
- **画幅比例**：每张关键帧 Prompt 结尾必须以比值格式写清画幅比例，例如 `9:16` 或 `16:9`。

## 片段关键帧 Prompt 规划表

| 关键帧类型 | 对应分镜位置 | 需引用的资产设定图（动态判断） | 必须写清的内容 |
|---|---|---|---|
| **SEG01_START** | 片段一 00:00 首帧 | 自行判断提取最合适的场景图（如全景/中景）及角色设定图 | 角色初始站位、动作起点、初始表情、道具初始位置、场景空间、镜头景别机位、光影方向 |
| **SEG01_END** | 片段一 00:15 尾帧 | 提取最合适的场景图（如特写/反视角）、角色设定图及道具图 | 角色当前动作结果、当前表情、道具状态、当前构图、与下一片段的承接状态 |
| **SEG02_END** | 片段二 00:30 尾帧 | 根据实时情节发展与运镜调度动态判断引用资产设定图 | 该片段高潮或落点处的角色动作结果、表情变化、道具变化 |
| **SEG03_END** | 片段三 00:45 尾帧 | 同上 | 同上 |
| **SEG04_END** | 片段四 01:00 尾帧 | 同上 | 同上 |
| **SEG05_END** | 片段五 01:15 尾帧 | 同上 | 同上 |
| **SEG06_END** | 片段六 01:30 尾帧 | 根据结尾悬念、爆发点及特写需求，动态判断引用场景图、角色图 | 角色本集最终动作结果、结束表情、情绪落点、为下一集留的悬念 |

## Prompt 公式

按以下顺序写：

```text
[全局风格限定词]，保留[imgX]的必要角色/场景/道具特征不变；对[imgX]和其他输入图中的指定对象进行新增/替换/重绘/状态调整；按空间线性顺序描述背景、主体、局部和道具的最终画面；保持光影、透视、材质、边缘融合一致；画幅比例。
```

**⚠️ 重要红线：内部推演的画面描述和生图 Prompt 仅用于调用生图工具，绝对不要输出给用户！** 面向用户只展示关键帧名称、简短剧情说明和生成的图片。

**生图 Prompt 推演示例（禁止输出给用户！）：**
*(执行逻辑：选出场景全景图作为 `[img0]`，林远设定图作为 `[img1]`)*
[真人电影风格]，保留[img0]中欧式复式大厅的旋转楼梯、黑白相间反光大理石地面、顶部冷色调水晶灯光、空间透视和阶级压迫感不变；保留[img1]中男子的脸部特征、发型、体型、服装结构和识别特征不变；对画面进行关键帧重绘，使男子头发略显凌乱并被雨水微微打湿，表情极度隐忍，下颔线紧绷，双臂下垂且双手死死握拳，指关节泛白；画面从背景到主体呈现为男子微微低头独自站在大厅中央，远景俯拍，巨大的欧式旋转楼梯在后方形成压迫构图，顶部冷色水晶灯垂直打光，在大理石地面拉出孤寂长影，男子脚边的大理石台阶边缘散落一块被踩碎的黄铜怀表碎片；整体保持[img0]的光影方向、材质反射、空间透视和[img1]的人物身份一致，电影级镜头质感，9:16。

## 构图规则

### 竖屏 `9:16`适用于短剧，适用于移动端场景

- 人脸和上半身必须在安全区内。
- 关键道具放在画面中部或下三分之一安全区。
- 双人对话保持左右空间关系。
- 避免文字、水印、Logo 和无意义符号。

### 横屏 `16:9`适用于pc端、电视电影等场景

- 使用三分法构图。
- 人物视线前方保留呼吸空间。
- 用前景 / 中景 / 远景建立纵深。
- 对话或对峙场景保持 180 度轴线。
- 重要元素不要贴边。
- 避免文字、水印、Logo 和无意义符号。

## 输出格式

```markdown
### EP01_SEG01_START
[简短剧情说明]
![EP01_SEG01_START](...)

### EP01_SEG01_END
[简短剧情说明]
![EP01_SEG01_END](...)

...（依次类推，直接展示全部 7 张生成的关键帧）

---
**验收与修改建议：**
以上是为您生成的本集片段关键帧图，请您查阅。如果您觉得有需要优化的地方，可以参考以下方向告诉我：
1.角色的长相、服装、发型在不同帧之间出现了明显的崩坏或漂移？
2.物理环境（如天气、白天/黑夜、光源方向）逻辑要更连贯？
3. 角色的动作幅度或画面构图（如 9:16）需要调整？
4. 如果有某一张关键帧抽卡不理想，您可以告诉我，我针对它单独重抽一次。
5. **进入下一步**：如果本集关键帧确认无误，我们可以继续生成下一集的关键帧。

## 连续性检查

交接前检查：

- 角色外观、发型、服装和身体状态一致。
- 道具在正确人物手中或正确场景位置。
- 场景光影、时间、天气和色温逻辑连续。
- 7 张关键帧与选定集数的片段匹配。

一集所有关键帧确认后，交接至 `prompt.md`，除非用户明确要求批量处理所有关键帧。
